CMake常用指令

Brandon Xing Lv1
  2025-03-10 17:58:12 2025-03-10 17:58:12 Created   2025-05-18 21:18:43 2025-05-18 21:18:43 Updated  

CMake 是一个跨平台的构建工具，用于管理 C/C++ 项目的编译过程。以下是 常用的 CMake 指令，涵盖项目配置、编译选项、依赖管理等方面：

1. 基础指令

（1）项目配置

指令 说明
cmake_minimum_required(VERSION 3.0) 指定 CMake 最低版本
project(MyProject VERSION 1.0 LANGUAGES CXX) 定义项目名称、版本和语言（CXX 表示 C++）
set(CMAKE_CXX_STANDARD 17) 设置 C++ 标准（如 C++11/14/17/20）
set(CMAKE_BUILD_TYPE Debug) 设置构建类型（Debug/Release/RelWithDebInfo）

（2）文件管理

指令 说明
include_directories(${PROJECT_SOURCE_DIR}/include) 添加头文件搜索路径（-I）
link_directories(${PROJECT_SOURCE_DIR}/lib) 添加库文件搜索路径（-L）
file(GLOB SOURCES "src/*.cpp") 使用通配符收集源文件（不推荐，可能导致增量编译失效）
aux_source_directory(. SRC_LIST) 收集当前目录下的源文件

2. 构建目标

（1）可执行文件

  • 生成可执行文件 my_app，依赖 main.cpp 和 utils.cpp。

（2）静态库

  • 生成静态库 libmy_lib.a（Linux）或 my_lib.lib（Windows）。

（3）动态库

  • 生成动态库 libmy_shared_lib.so（Linux）或 my_shared_lib.dll（Windows）。

3. 依赖管理

（1）链接库

  • 链接 pthread 库到 my_app（PRIVATE 表示仅当前目标使用）。

（2）查找系统库

  • 如果找不到库，REQUIRED 会报错。

（3）子目录管理

  • 常用于模块化项目。

4. 条件控制

（1）if 语句

  • 常用条件：
    • WIN32（Windows）
    • UNIX（Linux/macOS）
    • APPLE（macOS）

（2）option 选项

  • 可通过 cmake -DUSE_GPU=OFF 关闭。

5. 安装与打包

（1）安装目标

  • 运行 make install 进行安装。

（2）生成配置文件

  • 用于生成带版本号或路径的头文件。

6. 调试与优化

（1）打印调试信息

  • 输出级别：
    • STATUS（普通信息）
    • WARNING（警告）
    • FATAL_ERROR（终止编译）

（2）编译器选项

  • 常用选项：
    • -O2（优化）
    • -g（调试符号）
    • -Wall（启用警告）

7. 高级用法

（1）FetchContent（下载依赖）

  • 适用于动态下载 Google Test 等依赖。

（2）ExternalProject（外部项目）

  • 适用于编译第三方库（如 muduo）。

总结

类别 常用指令
项目配置 project(), cmake_minimum_required()
目标管理 add_executable(), add_library()
依赖管理 target_link_libraries(), find_package()
条件控制 if(), option()
安装打包 install(), configure_file()
调试优化 message(), set(CMAKE_CXX_FLAGS)
